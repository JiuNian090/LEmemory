# 云开发环境配置指南

## 📋 前置条件

1. 确保已在微信开发者工具中开通云开发
2. 确保环境ID已正确配置在 `app.ts` 中

## 🔧 配置步骤

### 1️⃣ 创建数据库集合

在微信开发者工具的「云开发」->「数据库」中，创建以下集合：

| 集合名称 | 说明 |
|---------|------|
| `cardGroups` | 卡牌组数据 |
| `cards` | 卡牌数据 |
| `favorites` | 收藏数据 |
| `studyRecords` | 学习记录 |
| `users` | 用户数据 |

### 2️⃣ 设置数据库权限

为每个集合设置权限规则，推荐设置为：
- **仅创建者可读写** 或 **所有用户可读，仅创建者可写**

### 3️⃣ 配置存储（可选）

如果需要图片存储功能，在「云开发」->「存储」中开通存储服务。

### 4️⃣ 创建云函数（可选）

如果需要获取用户openid，可以创建 `user_login` 云函数：

```javascript
// cloudfunctions/user_login/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  return {
    openid: wxContext.OPENID,
  }
}
```

然后在微信开发者工具中上传并部署云函数。

## 🚀 使用说明

### 首次使用

1. 确保云开发环境已开通并配置好
2. 在小程序中点击「新建卡牌组」创建第一个卡牌组
3. 数据会自动保存到云数据库

### 数据备份

云数据库中的数据会自动保存，建议定期在云开发控制台导出数据备份。

## ⚠️ 注意事项

1. **环境ID**：确保 `app.ts` 中的环境ID与你的云开发环境一致
2. **权限设置**：正确设置数据库权限，避免数据泄露
3. **免费额度**：注意云开发的免费额度限制

## 🔍 故障排查

### 问题1：提示 "Env Not Exists"

**解决方案**：
- 检查环境ID是否正确
- 确认云开发环境已开通
- 确认环境未过期或欠费

### 问题2：无法写入数据

**解决方案**：
- 检查数据库集合是否已创建
- 检查数据库权限设置
- 确认网络连接正常

### 问题3：云函数调用失败

**解决方案**：
- 确认云函数已上传部署
- 检查云函数代码是否正确
- 查看云开发控制台的日志

## 📞 技术支持

如有问题，请查看微信云开发官方文档或联系技术支持。
